# Incident Management

## Overview

Effective incident management involves immediate response, thorough investigation, and implementation of corrective actions to prevent recurrence. Every incident is a learning opportunity.

---

## Incident Classification

### OSHA Recordability

| Classification | Criteria |
|----------------|----------|
| **First Aid** | Treatment beyond first aid not required |
| **Recordable** | Medical treatment, restricted work, lost time |
| **Lost Time (LTI)** | Days away from work |
| **DART** | Days Away, Restricted, or Transfer |
| **Fatality** | Death resulting from work incident |

### First Aid vs. Medical Treatment

| First Aid Only | Medical Treatment |
|----------------|-------------------|
| Adhesive bandages | Sutures, staples |
| Butterfly closures | Prescription medications |
| Non-prescription medication (single dose) | Multiple doses of any medication |
| Wound cleaning | Wound requiring treatment beyond cleaning |
| Hot/cold packs | Physical therapy |
| Non-rigid support | Rigid support (casts, splints) |
| Eye wash | Eye treatment beyond irrigation |

### Incident Categories

| Category | Examples |
|----------|----------|
| **Personal injury** | Cuts, burns, fractures |
| **Illness** | Heat stress, chemical exposure |
| **Near miss** | Close call, no injury |
| **Property damage** | Equipment damage |
| **Environmental** | Spills, releases |
| **Fire/explosion** | Any fire event |
| **Well control** | Kick, blowout |
| **Security** | Theft, threats |

```

├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   1. SECURE THE SCENE                                          │
│      - Ensure safety of responders                              │

│                │                                                │
│                ▼                                                │
│   2. PROVIDE FIRST AID                                         │
│      - Assess injured person(s)                                 │
│      - Administer appropriate first aid                         │
│      - Call for medical assistance if needed                    │
│                │                                                │
│                ▼                                                │
│   3. NOTIFY                                                     │
│      - Supervisor/Company Man                                   │

│                                                                 │

```

### Notification Timeframes

| Incident Type | Notification |
|---------------|--------------|
| **Fatality** | Immediately, OSHA within 8 hours |
| **Hospitalization** | OSHA within 24 hours |
| **Amputation/eye loss** | OSHA within 24 hours |
| **LTI** | Same day to company |
| **Recordable** | Same day to company |
| **Significant near miss** | Same day to company |
| **Environmental release** | Per regulatory requirements |

## Incident Investigation

### Investigation Purpose

1. **Find root causes** - Not just symptoms
2. **Prevent recurrence** - Fix the system
3. **Not assign blame** - Focus on facts
4. **Learn and improve** - Share lessons

### Investigation Team

| Role | Responsibility |
|------|----------------|
| **Lead investigator** | Overall investigation |
| **HSE representative** | Technical guidance |
| **Supervisor** | Operational knowledge |
| **Employee representative** | Worker perspective |
| **Subject matter expert** | Technical expertise |

### Evidence Collection

| Type | Examples |
|------|----------|
| **Physical** | Equipment, materials, scene |
| **Documentary** | Procedures, permits, logs |
| **Testimonial** | Witness statements |
| **Environmental** | Weather, lighting, noise |
| **Photographic** | Scene photos, video |

### Witness Interviews

| Principle | Application |
|-----------|-------------|
| **Interview separately** | Prevent influence |
| **Interview promptly** | Fresh memories |
| **Open-ended questions** | "Tell me what happened" |
| **Non-judgmental** | Encourage honesty |
| **Document accurately** | Record statements |

## Root Cause Analysis

### Causal Factors

```
                    INCIDENT
                       │
           ┌───────────┼───────────┐
           │           │           │
           ▼           ▼           ▼
       Immediate    Basic      Root
       Causes       Causes     Causes
           │           │           │
           │           │           │
    ┌──────┼──────┐   │     ┌─────┼─────┐
    │      │      │   │     │     │     │
 Unsafe  Unsafe   │   │  Systems People Equip
 Acts    Conds    │   │     │     │     │
                  │   │     │     │     │
              Why?│Why?│  Why?│  Why?│  Why?
                  │   │     │     │     │
              ...continue asking "Why"...
```

### 5 Whys Technique

**Example**:
1. **What happened?** Worker burned hand
2. **Why?** Touched hot pipe
3. **Why?** Didn't know pipe was hot
4. **Why?** No warning sign or barricade
5. **Why?** No procedure for hot work area marking
6. **Root Cause**: Inadequate procedure for marking hot surfaces

### Common Root Cause Categories

| Category | Examples |
|----------|----------|
| **Procedures** | Absent, inadequate, not followed |
| **Training** | Insufficient, not current |
| **Communication** | Unclear, not provided |
| **Equipment** | Design flaw, maintenance failure |
| **Management** | Inadequate supervision, resources |
| **Human factors** | Fatigue, distraction, complacency |

## Corrective Actions

### SMART Corrective Actions

| Attribute | Meaning |
|-----------|---------|
| **S**pecific | Clear what needs to be done |
| **M**easurable | Can verify completion |
| **A**ssignable | Person responsible identified |
| **R**ealistic | Can actually be done |
| **T**ime-bound | Deadline established |

### Corrective Action Hierarchy

| Priority | Action Type | Example |
|----------|-------------|---------|
| **1** | Eliminate hazard | Remove the task |
| **2** | Engineering control | Guard, barrier |
| **3** | Administrative control | Procedure, training |
| **4** | PPE | Last resort |

### Tracking Corrective Actions

| Element | Requirement |
|---------|-------------|
| **Action item** | Specific description |
| **Owner** | Person responsible |
| **Due date** | Realistic deadline |
| **Status** | Open, in progress, complete |
| **Verification** | Evidence of completion |

## Near Miss Reporting

### Why Report Near Misses?

- Same causes as actual injuries
- Opportunity to prevent injuries
- Indicator of system weaknesses
- Shows engagement of workforce

### Near Miss Program Elements

| Element | Purpose |
|---------|---------|
| **Easy reporting** | Cards, app, verbal |
| **No punishment** | Encourage reporting |
| **Follow-up** | Show action taken |
| **Recognition** | Thank reporters |
| **Trending** | Identify patterns |

### Near Miss vs. Incident

| Near Miss | Incident |
|-----------|----------|
| Potential for injury | Actual injury |
| No harm occurred | Harm occurred |
| Same investigation value | Required investigation |
| May be underreported | Always reported |

## Incident Report Contents

### Required Information

| Section | Content |
|---------|---------|
| **When** | Date, time |
| **Where** | Location, specific area |
| **Who** | Persons involved, witnesses |
| **What** | Description of incident |
| **Why** | Contributing factors |
| **Classification** | Type, severity |
| **Immediate actions** | What was done |
| **Corrective actions** | Prevention measures |

### Report Quality Criteria

| Criterion | Description |
|-----------|-------------|
| **Accurate** | Facts verified |
| **Complete** | All information included |
| **Timely** | Completed promptly |
| **Objective** | No blame, just facts |
| **Clear** | Easy to understand |

## Learning from Incidents

### Lessons Learned Process

```
Investigate → Document → Analyze → Share → Apply
     │            │          │        │        │
     │            │          │        │        └── Implement changes
     │            │          │        └── Communicate to others
     │            │          └── Identify improvements
     │            └── Capture in report
     └── Determine causes
```

### Sharing Lessons Learned

| Method | Audience |
|--------|----------|
| **Safety meeting** | Immediate crew |
| **Safety alert** | All locations |
| **Company communication** | Organization-wide |
| **Industry sharing** | IADC, API, peers |
